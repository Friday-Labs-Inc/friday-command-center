Mark 1 — distributed modular
rover architecture

An offline-first, modular autonomous research rover. Five compute modules, three OS images, three transport domains, one signed command boundary. Built for agriculture, forestry, environmental research, and non-destructive surveillance — never weapons.

01

System overview

Each major subsystem owns its own compute and control layer, so the rover never depends on a single computer for everything. The Core Hub is the brain, not the bottleneck: heavy sensor work goes to the Lab Deck, communication to the Telemetry Gateway, movement to the Locomotion Unit, and drone operations to the Aerial Bay.

Communication architecture

Three distinct transport domains. No DDS ever crosses the Core ↔ Telemetry boundary, and no unsigned command ever crosses the external one.

DomainEndpointsTransportWire format
Robotics (internal) Core Hub ↔ Locomotion / Spark Bay ESP32s · Core Hub ↔ Lab Deck ROS 2 DDS · USB-serial micro-ROS (921600+ baud) friday_msgs (bounded, semver header)
Internal bridge Core Hub ↔ Telemetry Gateway MQTT over Ethernet (static IPs, internal subnet) Ed25519-signed CBOR envelopes
External boundary Telemetry Gateway ↔ Command Center (EMQX) MQTT 5 · TLS 1.3 · per-rover mTLS · radio failover Ed25519-signed CBOR envelopes

06

Security boundary

Every command crossing the external boundary is an Ed25519-signed, CBOR-encoded envelope with a monotonic nonce and an expiry. A command is accepted only if all checks hold — anything else is rejected with a category: never executed, never queued, never an exception.

Key placement

The rover's private key and the operator allowlist live only on the Core Hub. The Telemetry Gateway relays opaque signed bytes — compromising the internet-facing node yields no signing capability. A malformed operators file degrades to an empty allowlist (all commands rejected), never a crash.

Replay protection

Per-sender monotonic nonce floor, persisted to an fsync'd file and committed before the ACK — a reboot cannot reopen a replay window. Rejected commands never advance the floor. Rover-originated telemetry (odometry, faults, ACKs) is signed with the rover key so spoofed telemetry is equally rejected.

Failure matrix

FailureDetectionBehavior
Core Hub dies Locomotion heartbeat loss · Gateway internal-MQTT heartbeat loss Locomotion ESP32 hardware watchdog fires safe-stop in ≤100 ms (independent of both Pis). Gateway reports "Core down" to CC; CC may power-cycle the Core Hub via load switch.
Telemetry Gateway dies Core Hub bridge heartbeat loss Mission continues if safe; internal MQTT traffic queues locally; no external comms; backup ESP32 LoRa beacon if configured.
Both Pis die Backup ESP32 failure-detection logic Emergency LoRa beacon "Mark 1 alive, awaiting recovery". Locomotion already in safe-stop. The backup ESP32 holds no command authority.
Locomotion loses command link Firmware watchdog (100 ms deadline) Stop motors safely, lock current state, report fault if possible.
Split-brain rule. Command authority is held via explicit leases from the module registry. On failover the new holder acquires a fresh lease epoch; stale-epoch commands are rejected, so two brains can never drive the same motors.
08

AI harness — autonomy interface

The harness is a socket. Whatever brain plugs into it — simple rules, Nav2, vision AI, or a full LLM agent — the rest of the system (safety, authority, comms) stays untouched. Every AI action becomes a MotionCommand through the same authority and safety pipeline. No bypass.

Autonomy levels set per-mission by the operator

L0 · ManualAI decides nothing; direct joystick
L1 · AssistedAI: obstacle avoidance, speed limits · Operator: direction, goals
L2 · SupervisedAI plans paths; operator approves each waypoint
L3 · AutonomousAI executes the mission; operator monitors, can always override

Brain upgrade path same harness for all

B1 · Rulesif/else behaviors on the Pi 4B
B2 · Nav2path planning + behavior trees on the Pi 4B
B3 · Vision AIYOLO-class detection on Coral TPU / Lab Deck
B4 · LLM agentnatural-language missions, cloud via 4G or edge
